xft v0.0.1 Architecture

Minimal Tensor Framework for Deep Learning

Core Philosophy

xft is a minimal, understandable deep learning framework built from scratch. We target AMD CPUs, NVIDIA GPUs, and Linux only. No abstraction layers. No magic. Just clean code you can fully understand.

Technology Stack

  • Frontend: Python 3.8+ for user API
  • Backend: C++ for tensor operations and autograd engine
  • GPU: CUDA for GPU kernels
  • Bindings: pybind11 for Python-C++ interface
  • Build: CMake for compilation, pip for installation
  • License: MIT

v0.0.1 Scope

This version implements basic automatic differentiation with 1D arrays:

  • Array class with requires_grad support
  • Operations: add, multiply (element-wise and scalar)
  • Computation graph construction during forward pass
  • Reverse-mode automatic differentiation via .backward()
  • Gradient accumulation in .grad attribute
  • Support for scalars and 1D arrays only (no matrices yet)

Target Example

By end of v0.0.1, this code must work:

x = xft.Array([2.0], requires_grad=True)
w = xft.Array([3.0], requires_grad=True)
y = w * x + xft.Array([1.0])
y.backward()
print(x.grad, w.grad)

Design Decisions

  • Memory: Row-major layout (C-style), float32 only
  • Graph: Array-centric (each array tracks its creator)
  • Operations: Naive C++ loops (no BLAS yet)
  • Mode: Eager execution only (CPU only)
  • Threading: Single-threaded, not thread-safe
  • Errors: C++ exceptions propagated to Python
